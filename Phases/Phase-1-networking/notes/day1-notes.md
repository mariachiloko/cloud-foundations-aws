# Day 1 – Project Setup & Git Workflow

## What I Did
- Created a GitHub repository for the AWS networking project
- Cloned the repository locally to work in VS Code
- Set up initial folder structure (docs, terraform, notes)
- Added and committed initial files to GitHub

## What I Learned
- GitHub is the source of truth for the project
- Cloning a repo allows me to work locally while staying connected to GitHub
- Changes are made locally, then committed and pushed to GitHub
- It’s important to avoid having multiple versions of the same project

## Key Concepts
- Repository: The main project stored in GitHub
- Clone: A local copy of the repository
- Commit: Saving changes with a message
- Push: Sending changes to GitHub

## What Was Confusing
- Difference between creating a local project vs cloning from GitHub
- Understanding how everything stays in sync

## Clarification
- Only one project should exist — the cloned repo
- All work should happen inside that folder
- Git tracks all changes made inside the repo

## Takeaway
I now understand how to properly set up a project using GitHub and how local development connects to a remote repository.
